# Plan 05-01: GridFlags Configuration

**Goal:** Configure appropriate GridFlags for each routing device type

**Requirements:** AEIN-04 (Partial)

**Research:** `.planning/phases/05-ae2-integration/05-RESEARCH.md`

---

## Overview

Currently, all routing devices (Controller, Terminal, Cable) use `GridFlags.REQUIRE_CHANNEL`. This is incorrect - only the Routing Controller should consume an AE2 channel. Terminals and cables should be channel-neutral.

This plan refactors the GridFlags configuration to:
- RoutingController: REQUIRE_CHANNEL (consumes 1 AE2 channel)
- RoutingTerminal: No flags (channel-neutral)
- RoutingCable: No flags (channel-neutral)

## Implementation

### 1. Refactor AEBaseRouterTile for Flexible Flag Configuration

**File:** `src/main/java/com/ae2channelrouter/tile/AEBaseRouterTile.java`

Current code sets REQUIRE_CHANNEL in the base constructor. Refactor to:
- Remove automatic flag setting from constructor
- Add abstract method `configureGridFlags()` to be implemented by subclasses
- Call configuration method during initialization

```java
public abstract class AEBaseRouterTile extends AENetworkInvTile {
    
    public AEBaseRouterTile() {
        super();
        this.getProxy().setIdlePowerUsage(1.0);
        // Don't set flags here - let subclasses configure
    }
    
    @Override
    public void onReady() {
        super.onReady();
        configureGridFlags(); // Let subclass set appropriate flags
        this.getProxy().onReady();
    }
    
    /**
     * Configure GridFlags for this tile.
     * Called during onReady() to set appropriate flags.
     * Subclasses must implement to set their specific flags.
     */
    protected abstract void configureGridFlags();
}
```

### 2. Update RoutingControllerTile with REQUIRE_CHANNEL

**File:** `src/main/java/com/ae2channelrouter/tile/RoutingControllerTile.java`

Add implementation of `configureGridFlags()`:

```java
@Override
protected void configureGridFlags() {
    // Controller requires channel to participate in AE2 network
    this.getProxy().setFlags(GridFlags.REQUIRE_CHANNEL);
}
```

The controller needs a channel because:
- It must participate in the AE2 network to detect other controllers
- It acts as a bridge between routing system and AE2
- Only consumes 1 channel regardless of terminal count

### 3. Update RoutingTerminalTile as Channel-Neutral

**File:** `src/main/java/com/ae2channelrouter/tile/RoutingTerminalTile.java`

Add implementation of `configureGridFlags()`:

```java
@Override
protected void configureGridFlags() {
    // Terminal is channel-neutral - doesn't consume AE2 channels directly
    // Channels come from controller's pool, not from AE2 network
    this.getProxy().setFlags(); // No flags = neutral
}
```

The terminal should be neutral because:
- It receives channels from the routing controller's pool
- Acts as a pure forwarder/distributor
- Doesn't need direct AE2 network participation for channel functionality

### 4. Update RoutingCableTile as Channel-Neutral

**File:** `src/main/java/com/ae2channelrouter/tile/RoutingCableTile.java`

Add implementation of `configureGridFlags()`:

```java
@Override
protected void configureGridFlags() {
    // Cable is channel-neutral - just passes routing messages
    this.getProxy().setFlags(); // No flags
}
```

The cable should be neutral because:
- Physical connection only, no AE2 network participation
- Carries routing protocol messages, not AE2 channels
- Pure transport layer

### 5. Verify No Other Tiles Need Updates

Check all tile entity classes:
- `AEBaseRouterTile` - Abstract base, no changes needed beyond refactor
- `RoutingControllerTile` - Update with REQUIRE_CHANNEL
- `RoutingTerminalTile` - Update with no flags
- `RoutingCableTile` - Update with no flags

## Testing

2. Verify controller consumes 1 channel from AE2 network
3. Verify controller appears in AE2 network statistics

### Test 2: Terminal Channel Neutrality
1. Place Routing Terminal next to AE2 cable
2. Verify terminal does NOT consume AE2 channels
3. Verify terminal still functions correctly

### Test 3: Cable Channel Neutrality
1. Place Routing Cable next to AE2 cable
2. Verify cable does NOT consume AE2 channels
3. Verify cable still routes messages correctly

### Test 4: Mixed Configuration
1. Place controller, terminal, and cable in same AE2 network
2. Verify total AE2 channel usage = 1 (only controller)
3. Verify all routing devices function correctly
